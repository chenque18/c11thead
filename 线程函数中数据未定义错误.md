



# 传递给线程函数&时的数据未定义错误
```cpp
#include <thread>
#include <iostream>
using namespace std;
void fun(int& a){
    a+=1;
}
int main() {
    thread t(fun, 1);//报错：C++ 不允许把“临时数据”传给“能修改它的引用”
    t.join();
    return 0;
}
```
然而
```cpp
int x = 1;
thread t(fun, x);  // 依然报错！
```
这就是 std::thread 的特殊机制：它会把传入的参数都当成“要传给线程函数的实参”，而不是“要传给线程函数的引用”。因此，std::thread 会把 x 复制一份，传给线程函数，而不是传 x 的引用。
如果想要传引用，可以使用 std::ref 包装一下：
```cpp
int x = 1;
thread t(fun, ref(x));  // 这样就可以了！
```

# 传递*或&指向局部变量的问题
```cpp
#include <thread>
#include <iostream>
using namespace std;
thread t;
void fun(int& a){
    a+=1;
}
void test(){
    int b=1;
    t=thread(fun, ref(b));
}
int main() {
    test();
    t.join();
    return 0;
}
```
虽然通过编译，但运行报错未定义，因为test()结束时，t=thread(fun, ref(b));可能还在运行，但b已经回收了，导致b野指针。  
解决的核心思路都是：保证变量的存活时间一定要长于线程的运行时间。
方法一：在局部变量销毁前，等待线程结束
```cpp
void test(){
    int b=1;
    t=thread(fun, ref(b));
    t.join(); // 等待线程结束，确保b还存在
}
```
方法二：把变量定义在生命周期更长的地方
```cpp
int b=1; // 定义在全局，生命周期贯穿整个程序
```

# 传递类的非静态成员函数时的局部对象销毁问题
在使用 `std::thread` 调用类的非静态成员函数时，需要注意语法和对象的生命周期：  

错误例子
```cpp
#include <iostream>
#include <thread>

class MyClass {
public:
    void func() {
        std::cout << " Thread " << endl;
    }
};
int main() {
    MyClass obj;
    
    std::thread t(&MyClass::func, &obj);
    //obj在主线程被销毁了，但线程t还在运行，导致未定义行为
    return 0;
}
``` 
解决方法:智能指针
```cpp
#include <iostream>
#include <thread>
#include <memory> // 引入智能指针头文件
using namespace std;
class MyClass {
public:
    void func() {
        cout << "Thread running" << endl;
    }
};

int main() {
    // 1. 使用 shared_ptr 在堆上创建对象
    //shared_ptr<类名> 变量名 = make_shared<类名>(构造函数参数);
    shared_ptr<MyClass> obj = make_shared<MyClass>();
    
    thread t(&MyClass::func, obj);
    t.join(); 
    
    return 0;
}

# 入口函数为类的成员私有函数
解决方法:友元函数
```cpp
#include <iostream>
#include <thread>
#include <memory>
using namespace std;
class MyClass {
    friend void f_func(); // 声明友元函数
private:
    void func() {
        cout << "Thread running" << endl;
    }
};

void f_func(){
    shared_ptr<MyClass> obj = make_shared<MyClass>();
    thread t(&MyClass::func, obj);
    t.join(); 
}
int main() {
    f_func();
    return 0;
}
